# Manual Analysis Process

## Overview

We categorized documents based on their test generation impact using a systematic approach involving independent coding and consensus building.

## Process Steps

### 1. Initial Categories
We started with initial categories based on our knowledge of three document types:
- **Bug-inducing** (from GitHub issues)
- **Standard use cases** (from API documentation) 
- **Unique/edge use cases** (from Q&As)

### 2. Open-coding Thematic Analysis
We performed open-coding thematic analysis and clustered codes through systematic taxonomy development to expand beyond the initial three categories.

### 3. Codebook Development
We built a concise codebook following established methodology (Guest et al., 2011) with clear definitions and examples for each category.

### 4. Independent Coding
Two participants independently applied the codebook across all documents:
- **Coder A**: Systematic review of documents with detailed classification rationale
- **Coder B**: Independent classification with supporting evidence
- **Isolation**: No communication between coders during coding phase

### 5. Consensus Building
Two additional participants were involved to resolve disagreements through consensus:
- All disagreement cases were systematically reviewed
- Consensus reached through structured discussion
- Final classifications documented with rationale

## Results

### Final Classification
- All documents successfully classified into 7 categories
- 100% consensus achieved through systematic discussion
- Complete documentation of all decisions and rationale

### Quality Measures
- Inter-rater reliability ensured through independent coding
- Disagreements resolved through structured consensus process
- Transparent methodology with complete audit trail

This systematic process ensures reliable and reproducible document classification results for understanding how different document types influence automated test generation.
